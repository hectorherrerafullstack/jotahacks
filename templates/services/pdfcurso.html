Proyecto Django desde cero

Esta guía está pensada para quienes quieren empezar con buen pie en el mundo de Django. Aquí encontrarás cada paso explicado como si comenzaras desde cero, con comandos exactos y comprobaciones. Gracias por apoyarme en este camino; estoy seguro de que eres capaz de construir un proyecto sólido y de empezar a desatar toda tu creatividad.

Capítulo 1 — Instalar Visual Studio Code (VS Code)

Qué estás haciendo: instalas el editor donde vas a escribir y organizar el proyecto.

Por qué: VS Code trae terminal integrada, extensiones y es gratis.

Cómo (clic a clic)

  • Abre code.visualstudio.com y descarga para tu sistema.
  • Windows: siguiente → aceptar → instalar. Marca Agregar al PATH.
  • macOS: arrastra VS Code a Aplicaciones.
  • Linux: instala .deb/.rpm o usa tu gestor de paquetes.
  • Abre VS Code. Deberías ver la pantalla de Get Started.
Errores comunes: en Linux puede faltar libX11 u otros paquetes. Instálalos con apt o dnf.

Capítulo 2 — Instalar extensiones de VS Code

Qué estás haciendo: añades herramientas para que VS Code entienda Python y Django.

Por qué: tendrás autocompletado, linting y ayudas específicas.

Cómo

  • Abre VS Code → Extensiones (Ctrl+Shift+X).
  • Instala: Python (Microsoft), Pylance, Django, EditorConfig, Prettier (opcional).
Comprobación: las extensiones muestran un ✔ verde al quedar instaladas.

Capítulo 3 — Instalar Python (GRATIS)

Qué estás haciendo: instalas el lenguaje que usa Django.

Por qué: sin Python, no se puede ejecutar Django.

Windows

  • Entra en python.org → Downloads → Windows.
  • Descarga Windows installer (64-bit) y marca Add Python to PATH.

Verificar

python --version
pip --version

Si falla, usa:

py -3 --version
py -3 -m pip --version
Nota: si se abre Microsoft Store al escribir python, desactiva los App Execution Aliases.

macOS

  • Descarga el instalador universal desde python.org → macOS y ejecuta.

Verificar

python3 --version
pip3 --version

Si aparece Python 2.x al escribir python, usa python3.

Linux (Ubuntu/Debian)

sudo apt update
sudo apt install -y python3 python3-pip python3-venv
python3 --version
pip3 --version

Capítulo 4 — Instalar Git

Qué estás haciendo: instalas el sistema de control de versiones.

Por qué: podrás guardar cambios, crear copias de seguridad y volver atrás.

Cómo

  • Windows: descarga en git-scm.com y sigue Next.
  • macOS: brew install git
  • Ubuntu/Debian: sudo apt install -y git

Verificar

git --version
(Opcional) Configura tu autor: 
git config --global user.name "Tu Nombre"
git config --global user.email "tu@email.com"

Capítulo 5 — Crear carpeta de trabajo

Qué estás haciendo: creas la carpeta que contendrá tu proyecto.

Por qué: tener todo el código en un lugar claro evita confusiones.

mkdir proyectos_django
cd proyectos_django

Capítulo 6 — Crear entorno virtual (venv)

Qué estás haciendo: creas una "mochila" para aislar dependencias.

Por qué: cada proyecto tendrá sus librerías sin chocar con otras.

python -m venv .venv
Resultado esperado: aparece una carpeta .venv en tu proyecto.

Capítulo 7 — Activar el entorno virtual

Qué estás haciendo: te "pones" la mochila para trabajar dentro de ella.

Windows (PowerShell)

.\.venv\Scripts\Activate.ps1

macOS / Linux

source .venv/bin/activate
Comprobación: verás (.venv) al inicio de tu línea de comandos.
Windows: si PowerShell bloquea la activación, ejecuta como admin: 
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned

Capítulo 8 — Instalar Django

Qué estás haciendo: instalas el framework web.

Por qué: trae herramientas listas para no empezar desde cero.

pip install django
python -m django --version
Comprobación: debe mostrarse la versión instalada (por ejemplo, 5.x).

Capítulo 9 — Crear proyecto Django

Qué estás haciendo: generas la estructura base (configuración, rutas principales, etc.).

django-admin startproject config .
Resultado: se crea la carpeta config/ y el archivo manage.py.

Capítulo 10 — Migraciones iniciales

Qué estás haciendo: preparas la base de datos con las tablas por defecto.

python manage.py migrate
Comprobación: verás mensajes de "Applying ... OK".

Capítulo 11 — Crear superusuario

Qué estás haciendo: creas el usuario administrador del panel de Django.

python manage.py createsuperuser

Sigue las preguntas de usuario y contraseña.

Capítulo 12 — Encender el servidor de desarrollo

Qué estás haciendo: pones el proyecto a correr localmente.

python manage.py runserver
Comprobación: abre http://127.0.0.1:8000/ y deberías ver la página de bienvenida de Django.

Para detenerlo, vuelve a la terminal y presiona Ctrl+C.

Capítulo 13 — Crear una aplicación (app)

Qué estás haciendo: creas un "barrio" dentro del proyecto con su propia lógica.

python manage.py startapp miapp

Abre config/settings.py y añade la app:

INSTALLED_APPS = [
    "django.contrib.admin",
    "django.contrib.auth",
    "django.contrib.contenttypes",
    "django.contrib.sessions",
    "django.contrib.messages",
    "django.contrib.staticfiles",
    "miapp",  # ← añade esto
]

Capítulo 14 — Crear la vista inicial

Qué estás haciendo: defines qué se devuelve cuando alguien visita tu sitio.

Edita miapp/views.py:

from django.shortcuts import render

def home(request):
    return render(request, "miapp/home.html")

Capítulo 15 — Conectar las URLs

Qué estás haciendo: creas las "señales de tráfico" para llegar a cada vista.

miapp/urls.py:

from django.urls import path
from .views import home

urlpatterns = [
    path("", home, name="home"),
]

config/urls.py:

from django.contrib import admin
from django.urls import path, include

urlpatterns = [
    path("admin/", admin.site.urls),
    path("", include("miapp.urls")),
]

Capítulo 16 — Plantilla Home (HTML)

Qué estás haciendo: creas el archivo que verá el usuario en el navegador.

Crea carpetas de plantillas:

mkdir -p miapp/templates/miapp

Archivo miapp/templates/miapp/home.html:

<!doctype html>
<html lang="es">
<head>
  <meta charset="utf-8">
  <title>Inicio | Mi App</title>
</head>
<body>
  <h1>Bienvenido a mi primera app Django</h1>
  <p>Si ves esto, todo funciona correctamente.</p>
</body>
</html>

Capítulo 17 — Ver resultado

Qué estás haciendo: compruebas el flujo completo funcionando.

python manage.py runserver

Abre http://127.0.0.1:8000/ y revisa que cargue la Home.
